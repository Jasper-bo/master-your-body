VitalPulse 健康管理项目展示

这是一个本地优先的健康管理 Web 原型，围绕注册登录、生成个性化计划、记录饮食、 记录训练、查看健康评分组成最小可用闭环。此页面把项目中的实际界面截图、模块关系和 当前实现状态整理成一个可直接在浏览器打开的展示站点。

Overview

项目想解决什么问题

项目的核心目标是把日常健康行为从分散记录变成一个可见闭环：先建立个人身体目标， 再持续记录饮食和训练，最后用仪表盘把投入转化为清晰反馈。

用户价值

30 秒

通过预设食物、油脂选项和动作库降低记录成本，让用户每天能快速完成关键数据输入。

产品形态

本地优先

当前运行应用位于 my-app/，默认使用本地 SQLite 保存健康数据。

核心闭环

5 步

注册登录、生成计划、记录饮食、记录训练、查看健康评分与趋势。

Flow

用户从第一次进入到每日使用的流程

下面这条链路对应当前代码已经实现的主路径。PRD 中提到的验证码和 AI 拍照识别在当前版本中尚未完整落地。

1. 注册 / 登录 手机号与密码进入系统，JWT 写入 cookie 与本地 token 存储。
2. 个性化计划 填写身高、体重、年龄、性别、运动频率和目标。
3. 饮食记录 选择主食、肉类、蔬菜和油脂，保存为今日餐食。
4. 训练记录 从动作库加入训练清单，一次提交到本地数据库。
5. 仪表盘反馈 聚合营养、运动、睡眠、饮水并生成健康评分。
A

首次使用

用户注册后会进入不可跳过的计划弹窗；没有档案时，仪表盘和饮食页会提示先完成个性化计划。

B

每日记录

饮食页保存餐食会更新今日摄入；训练页保存 session 会自动同步运动打卡。

C

结果反馈

仪表盘计算 BMI、BMR、TDEE、营养目标完成度、健康评分和最近 7 天宏量营养趋势。

Dashboard

首页 / 仪表盘模块

仪表盘是项目的结果页，负责把用户当天的身体指标、饮食进度、运动情况和健康评分统一展示。

  • BMI、BMR、TDEE 与今日完成率。
  • 蛋白质、碳水、脂肪、热量目标进度。
  • 健康评分按营养、运动、睡眠、饮水拆分。
  • 最近 7 天宏量营养实际值与目标趋势。

Nutrition

饮食与营养模块

当前代码主链路以手动录入为准：用户从预设食物分类中选择食物、填写克数、选择油脂用量并保存当前餐食。

  • 主食类、肉类、蔬菜类和油脂用量四类输入。
  • 实时计算本餐热量与三大营养素。
  • 保存后刷新今日餐食和摄入进度。
  • 截图中出现的 AI 上传区属于产品方向，当前实现仍以手动路径为稳定基础。

Training

每日训练模块

训练模块围绕动作库和今日训练清单工作。用户按部位筛选动作，设置重量、组数、次数或有氧时长，再统一提交。

  • 胸部、背部、肩部、腿部、有氧分类。
  • 支持力量训练和有氧训练不同参数。
  • 展示今日目标、昨日回顾和本周统计。
  • 提交训练后更新今日记录和仪表盘运动打卡。

Settings

系统设置模块

设置页当前主要是信息展示和退出入口，是后续本地数据说明、备份导出、隐私策略等能力的承载位置。

  • 展示应用名称、版本和部署模式。
  • 保留更新日志、隐私政策、登出入口。
  • 当前仍偏信息框架，完整控制面板属于后续工作。

Auth & Onboarding

认证与个性化计划

注册和登录采用手机号加密码。注册成功后，用户必须完成个人计划，系统据此生成每日营养目标和身体指标。

注册 / 登录

  • POST /api/auth/register：创建账号，返回 access token 和 refresh token。
  • POST /api/auth/login：校验密码，失败次数过多会临时锁定账号。
  • POST /api/auth/refresh：刷新 access token。
  • POST /api/auth/logout：清理会话。

计划字段

  • 身高、体重、年龄、性别。
  • 运动频率：久坐、轻度、中度、高强度。
  • 健身目标：减脂、维持、增肌。
  • 后端计算 BMI、BMR、TDEE 和每日宏量营养目标。

API & Data

接口与数据如何支撑模块

项目采用 Next.js App Router，把页面与 API Routes 放在同一应用中；服务端逻辑集中在 my-app/src/lib/server，数据通过 Prisma 写入 SQLite。

仪表盘接口

  • GET /api/dashboard：聚合身体指标、营养、训练、清单与趋势。
  • 健康评分由服务端计算并 upsert 到 HealthScore。
  • 评分由营养、运动、睡眠、饮水四项组成，每项 25 分。

饮食接口

  • GET /api/nutrition/today：今日摄入、剩余热量和餐食列表。
  • GET /api/nutrition/food-categories：系统预设食物。
  • POST /api/nutrition/meals：保存餐食。
  • DELETE /api/nutrition/meals/[id]：删除餐食。

训练接口

  • GET /api/training/exercises：系统动作库。
  • POST /api/training/quick-log：快速记录训练。
  • GET /api/training/today：今日训练明细。
  • GET /api/training/weekly-stats：本周训练统计。

核心数据表

  • User、UserProfile、RefreshToken。
  • DailyNutrition、MealRecord、MealFood。
  • WorkoutRecord、训练动作与分类表。
  • HealthChecklist、HealthScore。
