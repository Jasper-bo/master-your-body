## 模块一：认证中心

> 对应页面：`/login`、`/register`

### 子模块 1.1：用户登录

#### 功能点 1.1.1：手机号登录

- **功能描述**：已注册用户通过手机号、密码及图形验证码登录系统。登录成功后返回 JWT Token，未登录用户访问任意受保护页面将被强制拦截。
- **用户场景**：用户每天打开应用查看健康数据时进行身份验证。
- **输入**：手机号（纯数字，长度 8-15 位，无具体号段校验）、密码（至少 6 位，无复杂度要求）、图形验证码（4 位数字）。
- **输出**：登录成功返回 JWT Access Token 与 Refresh Token，跳转至 `/dashboard`；失败返回具体错误信息（账号不存在 / 密码错误 / 验证码错误）。
- **优先级**：P0

#### 功能点 1.1.2：图形验证码获取与刷新

- **功能描述**：后端生成 4 位数字图形验证码图片，每次打开登录页或注册页时自动获取并展示。用户可点击图片手动刷新验证码。
- **用户场景**：用户需要输入验证码完成登录或注册，防止自动化攻击。
- **输入**：页面加载自动请求，或用户点击验证码图片触发刷新。
- **输出**：Base64 编码的 PNG 验证码图片及对应 session ID。
- **优先级**：P0

#### 功能点 1.1.3：登录态保持与登出

- **功能描述**：JWT Access Token 过期时通过 Refresh Token 自动刷新；用户在设置页主动点击登出时，清除本地及服务器端所有会话数据。
- **用户场景**：保持长期登录状态的便捷性与安全性；用户切换账号或退出应用。
- **输入**：Refresh Token（自动刷新时）；登出指令（主动退出时）。
- **输出**：新的 Access Token；清除所有会话并跳转至 `/login`。
- **优先级**：P0

### 子模块 1.2：用户注册

#### 功能点 1.2.1：手机号注册

- **功能描述**：新用户通过手机号、密码、确认密码及图形验证码创建 VitalPulse 账户。注册成功后立即触发个性化计划弹窗，不可跳过。
- **用户场景**：首次使用系统的用户需要创建个人账号以保存健康数据。
- **输入**：手机号（纯数字，8-15 位）、密码（至少 6 位）、确认密码、图形验证码。
- **输出**：注册成功，自动弹出个性化计划弹窗；失败返回具体错误信息（手机号已存在 / 密码不一致 / 验证码错误）。
- **优先级**：P0

### 子模块 1.3：个性化计划弹窗

#### 功能点 1.3.1：首次注册强制信息采集

- **功能描述**：新用户注册成功后立即在注册页弹出个性化计划弹窗，**不可关闭/跳过，必须填完所有字段才能进入系统**。后端根据采集数据自动计算 BMI、BMR（Mifflin-St Jeor）、TDEE、每日热量目标及三大营养素分配目标。饮水目标默认 2000ml，睡眠目标默认 7.5 小时。
- **用户场景**：新用户注册后完善身体数据，系统据此生成个性化营养与训练方案。
- **输入**：
  - 身高（cm）
  - 体重（kg）
  - 年龄（整数，10-100）
  - 性别（男 / 女 / 其他）
  - 运动频率（4 档单选）：
    1. 久坐不动
    2. 每周轻度运动 1-2 天
    3. 每周中度运动 3-5 天
    4. 每周剧烈运动 6-7 天
  - 健身目标（3 档单选）：减脂 / 维持 / 增肌
- **输出**：保存用户档案，自动计算并持久化：BMI、BMR、TDEE、每日热量目标(kcal)、蛋白质目标(g)、碳水目标(g)、脂肪目标(g)、饮水目标 2000ml、睡眠目标 7.5h；完成后自动进入 `/dashboard`。
- **优先级**：P0

### 子模块 1.4：全局认证守卫

#### 功能点 1.4.1：未登录访问拦截

- **功能描述**：未登录用户访问 `/dashboard`、`/nutrition`、`/training`、`/settings` 等任意受保护页面时，强制跳转至 `/login`。
- **用户场景**：防止未授权用户访问个人健康数据。
- **输入**：用户访问受保护路由。
- **输出**：HTTP 302 重定向至 `/login`。
- **优先级**：P0

## 模块二：仪表盘（主页）

> 对应页面：`/dashboard`

### 子模块 2.1：欢迎与日期区

#### 功能点 2.1.1：日期与动态欢迎语

- **功能描述**：展示当前日期，并根据系统时间段显示动态欢迎语：早上好（06:00-11:59）、下午好（12:00-17:59）、晚上好（18:00-22:59）、夜深了（23:00-05:59）。
- **用户场景**：用户打开主页时获得亲切的使用体验。
- **输入**：系统当前时间。
- **输出**：格式化日期 + 动态欢迎语文本。
- **优先级**：P1

### 子模块 2.2：每日营养卡片

#### 功能点 2.2.1：三大营养素进度条

- **功能描述**：以三条可视化进度条分别展示蛋白质、碳水化合物、脂肪的当前累计摄入量与每日目标值对比。
- **用户场景**：用户快速查看今日三大营养素的完成进度。
- **输入**：用户当日累计摄入数据（蛋白质 g、碳水 g、脂肪 g）、个人营养目标值。
- **输出**：三项带颜色区分的进度条及百分比（当前值 / 目标值）。
- **优先级**：P0

### 子模块 2.3：身体指标速览

#### 功能点 2.3.1：BMI 实时计算与展示

- **功能描述**：基于用户最新身高与体重实时计算 BMI 指数，并展示对应健康等级。
- **用户场景**：用户快速了解自身体重健康状况。
- **输入**：用户最新身高（cm）、体重（kg）。
- **输出**：BMI 数值（保留 1 位小数）、等级标签（偏瘦 <18.5 / 正常 18.5-23.9 / 偏胖 24-27.9 / 肥胖 ≥28）。
- **优先级**：P0

#### 功能点 2.3.2：BMR 展示

- **功能描述**：基于用户性别、年龄、身高、体重展示基础代谢率（BMR），采用 Mifflin-St Jeor 公式计算。
- **用户场景**：用户了解自身静息状态下每日最低能量消耗。
- **输入**：性别、年龄、身高、体重。
- **输出**：BMR 数值（kcal/天）。
- **优先级**：P0

#### 功能点 2.3.3：今日完成率

- **功能描述**：综合计算今日饮食与训练目标的加权完成百分比，展示在仪表盘核心区域。
- **用户场景**：用户一目了然今日整体健康计划执行进度。
- **输入**：今日摄入完成度、今日训练计划完成度。
- **输出**：完成率百分比。
- **优先级**：P1

### 子模块 2.4：健康评分清单

#### 功能点 2.4.1：三项打卡状态管理

- **功能描述**：用户对三项健康行为进行每日打卡管理：水分补充（手动输入 ml）、睡眠质量（手动输入 h）、运动锻炼（训练完成后自动触发打卡）。
- **用户场景**：用户记录当日健康行为习惯完成情况。
- **输入**：饮水量（ml）、睡眠时长（h）、运动完成状态（系统自动）。
- **输出**：更新后的打卡状态，实时同步至总评分。
- **优先级**：P0

#### 功能点 2.4.2：总评分环形图

- **功能描述**：基于健康评分算法计算百分制总评分，并以环形图可视化展示，附带等级评定。总分由营养完成度、运动锻炼、睡眠质量、水分补充四项构成。
- **用户场景**：用户获得直观的每日健康行为量化评分。
- **输入**：营养完成度（25 分）+ 运动锻炼（25 分）+ 睡眠质量（25 分）+ 水分补充（25 分）。
- **输出**：总分（0-100）、等级（A≥90 / B≥75 / C≥60 / D<60）、环形图填充比例。
- **优先级**：P0

### 子模块 2.5：营养趋势分析

#### 功能点 2.5.1：7 日营养宏量趋势图

- **功能描述**：以折线图展示最近 7 天蛋白质、碳水、脂肪的每日实际摄入量（3 条实线）与对应目标值（3 条虚线）的对比趋势。
- **用户场景**：用户回顾近期营养摄入波动，调整饮食策略。
- **输入**：过去 7 天每日实际摄入数据、每日营养目标值。
- **输出**：六条折线图（3 条实线 + 3 条虚线），X 轴为日期，Y 轴为克数。
- **优先级**：P1

### 子模块 2.6：个性化计划入口

#### 功能点 2.6.1：浮动更新计划按钮

- **功能描述**：仪表盘右下角浮动"更新计划"按钮，老用户可点击修改身体数据（身高、体重、运动频率、健身目标），系统重新计算 BMI、BMR、TDEE 及每日营养目标。
- **用户场景**：老用户因身体变化或目标调整需要更新个性化方案。
- **输入**：身高（cm）、体重（kg）、运动频率（4 档）、健身目标（减脂/维持/增肌）。
- **输出**：重新计算并更新 BMI、BMR、TDEE、每日热量目标、三大营养素目标。
- **优先级**：P0

## 模块三：饮食与营养

> 对应页面：`/nutrition`

### 子模块 3.1：AI 食物识别

#### 功能点 3.1.1：AI 拍照识别食物

- **功能描述**：用户上传食物照片，前端通过 multipart/form-data 上传至本地后端，后端转发 DeepSeek API 进行分析。**不保存原始图片，分析完成后立即丢弃**。DeepSeek 返回食物名称、蛋白质(g)、碳水(g)、脂肪(g)，热量由前端计算（蛋白×4 + 碳水×4 + 脂肪×9）。
- **用户场景**：用户外出就餐或在家做饭，不方便手动查找时使用拍照快速记录。
- **输入**：图片文件（multipart/form-data，JPG/PNG，最大 10MB）。
- **输出**：识别结果（食物名称、蛋白质 g、碳水 g、脂肪 g）、前端计算热量（kcal）。
- **优先级**：P0

#### 功能点 3.1.2：AI 识别结果确认与修正

- **功能描述**：识别结果显示在饮食页上传区旁，用户可对蛋白质、碳水、脂肪三个数值进行人工修改后确认使用。
- **用户场景**：AI 识别存在误差，用户需要人工校正后保存。
- **输入**：修正后的蛋白质(g)、碳水(g)、脂肪(g)。
- **输出**：修正后的营养素数据用于后续餐食录入。
- **优先级**：P0

### 子模块 3.2：今日摄入概览

#### 功能点 3.2.1：今日摄入数据统计

- **功能描述**：汇总展示今日已记录的所有餐食的热量及三大营养素总量。
- **用户场景**：用户查看截止当前的所有饮食摄入累计数据。
- **输入**：当日所有已确认的饮食记录。
- **输出**：总热量（kcal）、总蛋白质（g）、总碳水（g）、总脂肪（g）。
- **优先级**：P0

#### 功能点 3.2.2：日摄入目标完成度总进度条

- **功能描述**：以单条总进度条展示今日热量摄入占每日热量目标的百分比，并显示剩余可摄入热量。
- **用户场景**：用户直观判断今日饮食是否超标或不足。
- **输入**：今日累计热量、每日热量目标。
- **输出**：进度条百分比、剩余可摄入热量（kcal）。
- **优先级**：P0

### 子模块 3.3：手动食物录入

#### 功能点 3.3.1：主食类录入

- **功能描述**：从预设主食库（5 种）中选择并录入主食类食物及克数。预设选项：糙米饭、全麦面包、红薯、燕麦、意面。
- **用户场景**：用户食用主食后手动记录。
- **输入**：食物选择（5 种之一）、克数（10-2000g）。
- **输出**：根据食物库每 100g 营养素换算实际摄入，添加到"当前餐食待确认列表"。
- **优先级**：P0

#### 功能点 3.3.2：肉类录入

- **功能描述**：从预设肉库（5 种）中选择并录入肉类食物及克数。预设选项：鸡胸肉、三文鱼、瘦牛肉、虾仁、豆腐。
- **用户场景**：用户食用蛋白质来源后手动记录。
- **输入**：食物选择（5 种之一）、克数（10-1000g）。
- **输出**：根据食物库每 100g 营养素换算实际摄入，添加到"当前餐食待确认列表"。
- **优先级**：P0

#### 功能点 3.3.3：蔬菜类录入

- **功能描述**：从预设蔬菜库（5 种）中选择并录入蔬菜类食物及克数。预设选项：西兰花、菠菜、生菜、西红柿、胡萝卜。
- **用户场景**：用户食用蔬菜后手动记录。
- **输入**：食物选择（5 种之一）、克数（10-1000g）。
- **输出**：根据食物库每 100g 营养素换算实际摄入，添加到"当前餐食待确认列表"。
- **优先级**：P0

#### 功能点 3.3.4：油脂用量录入

- **功能描述**：以 4 档单选方式记录烹饪油脂用量。选项：无油（0g）、少油（5g）、中等（10g）、多油（20g）。
- **用户场景**：用户估算菜品油脂用量，更精确追踪脂肪摄入。
- **输入**：油脂档次单选（4 档之一）。
- **输出**：换算为脂肪克数（脂肪 × 9 kcal/g 计入热量），添加到"当前餐食待确认列表"。
- **优先级**：P1

#### 功能点 3.3.5：确认并记录当前餐食

- **功能描述**：用户完成一餐的食物选择后，选择餐别（早餐/午餐/晚餐/加餐，默认按当前时间自动推断）并确认，将当前餐食正式保存到当日饮食记录。
- **用户场景**：用户一餐结束后统一确认录入。
- **输入**：当前餐食待确认列表中的所有食物项、餐别（早/午/晚/加餐）。
- **输出**：写入数据库，更新今日摄入概览，`/dashboard` 数据同步刷新。
- **优先级**：P0

## 模块四：每日训练

> 对应页面：`/training`

### 子模块 4.1：训练概览

#### 功能点 4.1.1：今日目标卡片

- **功能描述**：展示"今日已计划/完成 X 次训练"、训练部位、预计总时长、当前完成进度。
- **用户场景**：用户进入训练页后快速了解今日训练安排及完成情况。
- **输入**：当日训练计划数据、已完成动作数 / 计划动作总数。
- **输出**：训练次数、训练部位标签、预计时长（分钟）、完成进度百分比。
- **优先级**：P0

#### 功能点 4.1.2：昨日训练回顾

- **功能描述**：展示昨日最后一次训练的动作列表及参数，供用户参考。
- **用户场景**：用户参考昨日训练强度，决定今日是否递增负荷。
- **输入**：昨日已完成的最后一次训练记录。
- **输出**：昨日训练动作列表（动作名、重量/时长、组数×次数 等参数）。
- **优先级**：P1

#### 功能点 4.1.3：本周统计

- **功能描述**：汇总本周训练数据，包括训练次数、总时长、同比上周增幅百分比。
- **用户场景**：用户评估本周训练频率和时长是否达标。
- **输入**：本周（周一至当日）已完成训练记录。
- **输出**：本周训练次数、总时长（分钟）、同比上周增幅百分比（上周为 0 时显示 N/A）。
- **优先级**：P1

### 子模块 4.2：动作库

#### 功能点 4.2.1：动作库列表（Bento 网格）

- **功能描述**：以 Bento 网格布局展示系统预设动作库，按部位分类筛选：胸部、背部、肩部、腿部、有氧。
- **用户场景**：用户浏览可用动作，选择今日要训练的内容。
- **输入**：部位筛选（全部 / 胸部 / 背部 / 肩部 / 腿部 / 有氧）。
- **输出**：动作卡片网格（动作名、所属部位）。
- **优先级**：P0

#### 功能点 4.2.2：力量训练动作添加

- **功能描述**：在动作库中选择力量训练动作后，输入重量（kg）、组数、次数，点击"+"添加到"今日训练清单"。
- **用户场景**：用户规划今日力量训练内容。
- **输入**：动作选择、重量（kg，0-500）、组数（1-20）、次数（1-100）。
- **输出**：添加到前端"今日训练清单"暂存区。

## 附录：核心业务规则

### A.1 BMI（身体质量指数）计算

```
BMI = 体重(kg) / (身高(m))²

等级划分：
- 偏瘦：< 18.5
- 正常：18.5 ≤ BMI < 24.0
- 偏胖：24.0 ≤ BMI < 28.0
- 肥胖：≥ 28.0
```

### A.2 BMR（基础代谢率）计算

采用 **Mifflin-St Jeor** 公式：

```
男性 BMR (kcal/天) = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄(岁) + 5
女性 BMR (kcal/天) = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄(岁) - 161
```

### A.3 TDEE（每日总能量消耗）计算

```
TDEE = BMR × 活动系数

活动系数（4 档）：
- 久坐不动（很少或不运动）：1.2
- 每周轻度运动 1-2 天：1.375
- 每周中度运动 3-5 天：1.55
- 每周剧烈运动 6-7 天：1.725
```

### A.4 每日营养目标计算

**热量目标：**

```
- 减脂目标：TDEE - (300 ~ 500) kcal
- 维持目标：TDEE
- 增肌目标：TDEE + (200 ~ 300) kcal
```

**三大营养素分配：**

```
蛋白质(g)：
- 增肌：体重(kg) × 2.0 ~ 2.2
- 减脂：体重(kg) × 1.6 ~ 2.0
- 维持：体重(kg) × 1.2 ~ 1.6

脂肪(g)：体重(kg) × 0.8 ~ 1.0

碳水(g)：(热量目标 - 蛋白质热量 - 脂肪热量) / 4
- 其中：蛋白质热量 = 蛋白质(g) × 4 kcal/g
- 脂肪热量 = 脂肪(g) × 9 kcal/g
```

**其他目标默认值：**

```
- 饮水目标：2000 ml / 天
- 睡眠目标：7.5 h / 天
```

### A.5 健康评分算法

```
总分 = 营养完成度（25分）+ 运动锻炼（25分）+ 睡眠质量（25分）+ 水分补充（25分）= 100分

计分规则：
- 营养完成度（25分）：当日热量摄入达到每日目标的 80%-120% 视为完成，得 25 分
- 运动锻炼（25分）：当日有完成训练记录视为完成，得 25 分（训练完成后自动触发打卡）
- 睡眠质量（25分）：当日记录睡眠时长达到目标值（默认 7.5h）的 80%-120% 视为完成，得 25 分
- 水分补充（25分）：当日记录饮水量达到目标值（默认 2000ml）的 80%-120% 视为完成，得 25 分

等级评定：
- A（优秀）：≥ 90 分
- B（良好）：≥ 75 分
- C（及格）：≥ 60 分
- D（需改善）：< 60 分
```

### A.6 食物热量换算

**AI 识别热量计算（前端计算）：**

```
热量(kcal) = 蛋白质(g) × 4 + 碳水(g) × 4 + 脂肪(g) × 9
```

**预设食物库换算（每 100g 基准）：**

```
实际摄入热量 = (食物每100g热量 kcal × 输入克数) / 100
实际摄入蛋白质 = (食物每100g蛋白质 g × 输入克数) / 100
实际摄入碳水 = (食物每100g碳水 g × 输入克数) / 100
实际摄入脂肪 = (食物每100g脂肪 g × 输入克数) / 100
```

**油脂热量换算：**

```
油脂热量 = 脂肪克数 × 9 kcal/g
```

### A.7 训练完成度计算

```
今日训练计划完成度 = (已完成动作数 / 计划动作总数) × 100%

本周统计：
- 训练次数 = 本周（周一 00:00 至周日 23:59）完成训练的天数
- 总时长 = 本周所有训练记录的预计时长累加
- 同比上周增幅 = (本周总时长 - 上周总时长) / 上周总时长 × 100%（上周为 0 时显示 N/A）

注：同一天多次训练保留独立记录展示，不合并组数。
```

### A.8 数据保留规则

```
- 饮食和训练明细记录保留 90 天后自动物理删除
```
